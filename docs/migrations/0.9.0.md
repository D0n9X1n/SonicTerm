# Migration: v0.9.0

The v0.9 modularization (issue #429) introduces:
1. **Trait seams** in `sonicterm-types` (`PtyTransport`, `Painter`,
   `WindowBackend`, `ClipboardBackend`, `FrameLike`) for the
   modularization pilot post-v0.9. **Additive** — no behavior change.
2. **`sonicterm-app-core`** — winit-agnostic state-machine crate (M6a).
   Currently a sketch; consumers migrate over post-v0.9.
3. **`#[deprecated(since = "0.9.0")]`** on every re-export from the
   façade crates `sonicterm-core` and `sonicterm-shared`. Removal target:
   **v1.1**.

## Façade → leaf mapping

### `sonicterm-core` (removal target: v1.1)

| Old import                          | New import                          |
|---|---|

| `use sonicterm_core::theme::...`    | `use sonicterm_cfg::theme::...`     |
| `use sonicterm_core::keymap::...`   | `use sonicterm_cfg::keymap::...`    |
| `use sonicterm_core::url_open::...` | `use sonicterm_cfg::url_open::...`  |

## Removal timeline

- **v0.9.0** — façades emit `#[deprecated]` warnings; no breakage.
- **v0.10.x – v1.0.x** — soft period; downstream consumers migrate.
  PRs in the modularization pilot move first-party uses off the façades.
- **v1.1.0** — `sonicterm-core` and `sonicterm-shared` are DELETED.
  Tag-owner is the sole writer of this file during the v1.1 release
  window.

## How to migrate a single file

```bash
# replace sonicterm_core::vt with sonicterm_vt::vt etc.
sed -i '' \
  -e 's|sonicterm_core::vt|sonicterm_vt::vt|g' \
  -e 's|sonicterm_core::grid|sonicterm_grid::grid|g' \
  -e 's|sonicterm_core::config|sonicterm_cfg::config|g' \
  -e 's|sonicterm_core::pty|sonicterm_io::pty|g' \
  <files>
```

Then add the corresponding leaf to the consuming crate's `Cargo.toml`
(`{leaf} = { workspace = true }`) and remove the `sonicterm-core` /
`sonicterm-shared` dep if it's the last user.
