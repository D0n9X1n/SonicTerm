# Migration: v0.9.0

The v0.9 modularization (issue #429) introduces:
1. **Trait seams** in `sonicterm-types` (`PtyTransport`, `Painter`,
   `WindowBackend`, `ClipboardBackend`, `FrameLike`) for the
   modularization pilot post-v0.9. **Additive** — no behavior change.
2. **`sonicterm-app-core`** — winit-agnostic state-machine crate (M6a).
   Currently a sketch; consumers migrate over post-v0.9.
3. **`#[deprecated(since = "0.9.0")]`** on every re-export from the
   façade crates `sonicterm-core` and `sonicterm-shared`. (These
   façades were deleted in v1.1 — see PR #469.)

## Façade → leaf mapping

| `use sonicterm_core::url_scan::...` | `use sonicterm_cfg::url_scan::...`  |
| `use sonicterm_core::grid::...`     | `use sonicterm_grid::grid::...`     |

| `use sonicterm_core::proc_info::...`| `use sonicterm_io::proc_info::...`  |

## Removal timeline

- **v0.9.0** — façades emit `#[deprecated]` warnings; no breakage.
- **v0.10.x – v1.0.x** — soft period; downstream consumers migrate.
  PRs in the modularization pilot move first-party uses off the façades.
- **v1.1.0** — `sonicterm-core` and `sonicterm-shared` were DELETED
  (PR #469).

## How to migrate a single file

```bash
# replace sonicterm_core::vt with sonicterm_vt::vt etc.
sed -i '' \
  -e 's|sonicterm_core::vt|sonicterm_vt::vt|g' \
  -e 's|sonicterm_core::grid|sonicterm_grid::grid|g' \
  -e 's|sonicterm_core::config|sonicterm_cfg::config|g' \
  -e 's|sonicterm_core::pty|sonicterm_io::pty|g' \
  <files>
```

Then add the corresponding leaf to the consuming crate's `Cargo.toml`
(`{leaf} = { workspace = true }`) and remove the `sonicterm-core` /
`sonicterm-shared` dep if it's the last user.

## Breaking: legacy `Sonic/sonic.toml` config path no longer auto-migrated

Per #469, the runtime shim that copied the pre-R3 config layout to the
new path has been removed. Users still on the OLD config path must
manually rename:

- **macOS** — `~/Library/Application Support/Sonic/sonic.toml` →
  `~/Library/Application Support/SonicTerm/sonicterm.toml`
- **Windows** — `%APPDATA%\Sonic\sonic.toml` →
  `%APPDATA%\SonicTerm\sonicterm.toml`
- **Linux** (dev) — `~/.config/sonic/sonic.toml` →
  `~/.config/sonicterm/sonicterm.toml`

If no SonicTerm config file is present at first launch, defaults apply
and the file is created on first save.

## M6d — legacy `sonicterm_app::run_with*` removed

M6d removes deprecated `sonicterm_app::run_with*` entry points
(`run`, `run_with`, `run_with_os_drag`, `run_with_os_drag_and_pending`,
`run_with_os_drag_pending_and_hook`,
`run_with_os_drag_pending_and_window_hook`). Consumers must use
`sonicterm_app::shell::{MacShell,WindowsShell}::new(machine).run()`,
constructing the `AppStateMachine` in the bin and handing it to the
shell builder.
