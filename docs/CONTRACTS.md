# docs/CONTRACTS.md

The **contract crate** is `crates/sonicterm-types/`. It owns the value
types and trait seams that every other crate depends on. A change to its
public API is a contract change and follows the deprecation protocol
below.

## Public surface (authoritative)

The machine-checked source of truth is
`crates/sonicterm-types/api-snapshot.txt`, generated by
`cargo public-api -p sonicterm-types --simplified`. The script
`tools/check-contract-docs.sh` fails the PR gate if the snapshot drifts
without an accompanying update.

### Value types
- `Cell` — one terminal cell (glyph + fg + bg + attrs).
- `Pos`, `Rect` — grid coordinates.
- `Action` — bindable user action enum (consumed by `sonicterm-cfg::keymap`
  and `sonicterm-app::app::keymap_dispatch`).
- `GlyphKey`, `HyperlinkId` — interning keys.
- `FrameModel` — renderer-agnostic frame description.

### Trait seams
- `PtyTransport` — read/write/resize/wait API a PTY backend must
  implement. Default impl in `sonicterm-io::pty::PtyHandle`.
- `Painter` (aka `FrameSink`) — what the app loop hands a `FrameModel`
  to. Default impl in `sonicterm-gpu`.
- `WindowBackend` — minimal abstraction over the windowing system.
  Default impl wraps `winit::window::Window`.
- `ClipboardBackend` — copy / paste indirection. Default impl in
  `sonicterm-app` uses `arboard`.

Each trait has a `crates/sonicterm-types/tests/contract_<trait>.rs`
contract test asserting object-safety, method signatures, and any
invariants the trait promises.

## Deprecation protocol (2-PR sequence)

Any breaking change to a `pub` item in `sonicterm-types` MUST go through
two PRs across two minor versions:

1. **PR-A** (lands at vX.Y) — add the new API alongside the old. Mark
   the old:
   ```rust
   #[deprecated(since = "X.Y.0", note = "use NewThing; will be removed in X.Y+2")]
   ```
   Add a `docs/migrations/X.Y.0.md` entry with before/after snippets.
2. **PR-B** (lands at vX.Y+2 or v(X+1).0, whichever is sooner) — remove
   the deprecated API. Must land during a release-tag PM window.

**Removal without a prior `#[deprecated]` shipped in a tagged release is
a CI-fail** regardless of the `contract-change` label on the PR.

Re-exports from the deprecated façades `sonicterm-core` and
`sonicterm-shared` follow the same protocol but the removal target is
v1.1 (see `docs/migrations/0.9.0.md`).

## Three self-enforcing gates

| Gate | File | Lands at | Flips to FAIL at |
|---|---|---|---|
| Public-API drift vs snapshot/docs | `tools/check-contract-docs.sh` | M1 | M8 |
| Landmine TOML self-test (globs + tests exist) | `crates/sonicterm-types/tests/contract_landmine_coverage.rs` | M2 | M8 |
| Every pub trait has a contract test | `crates/sonicterm-types/tests/contract_traits_have_tests.rs` | M3 | M8 |

Between landing and flip, the gates run as **warnings** so the surface
can stabilize. After M8 they block merges.

---

## How to make a contract change

1. Open an issue with the `contract-change` label. Include before/after
   API snippets and rationale.
2. Land PR-A: new API + `#[deprecated]` on the old + migration doc.
3. Re-run `cargo public-api -p sonicterm-types --simplified > crates/sonicterm-types/api-snapshot.txt`
   and commit the regenerated snapshot in the same PR.
4. Wait one minor release. Land PR-B with the removal.
5. PR-B body MUST link the PR-A deprecation commit and the migration
   doc that shipped with PR-A.
