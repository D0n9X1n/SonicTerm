# sonicterm-types

## Purpose
The **contract crate**. Zero-dep value types + trait seams that every
other crate depends on. Public API changes affect the whole workspace and
should be paired with tests and a snapshot update when applicable.

## Public surface
- Value types: `Cell`, `Pos`, `Rect`, `Action`, `GlyphKey`, `HyperlinkId`, `FrameModel`
- Trait seams: `PtyTransport`, `Painter`, `WindowBackend`, `ClipboardBackend` (M4)
- Snapshot of truth: `api-snapshot.txt` (generated by `cargo public-api`)

## Land-mines specific to this crate
None — but every change here ripples to every consumer. Treat as
contract.

## Test gate (local)
```bash
cargo build -p sonicterm-types
```

The crate ships three self-enforcing meta-tests:
- `contract_landmine_coverage.rs` — every entry in landmines.toml resolves
- `contract_traits_have_tests.rs` — every pub trait has a `contract_<trait>.rs`
- per-trait `contract_<trait>.rs` files asserting shape + invariants

## Common pitfalls
- Adding a pub trait should include a focused contract/unit test.
- Removing a public item is a breaking API change.
- Adding `derive(Serialize)` is a contract-API change too — update snapshots when used.

## Owning PM(s)
- Primary: either (cross-platform pure-data)
- Hot-file: yes — every consumer rebuilds

## Cross-references
- Consumed by: every other crate in the workspace
